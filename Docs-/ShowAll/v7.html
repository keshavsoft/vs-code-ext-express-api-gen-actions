How ShowAll Endpoint Gets Generated

Late at night, a developer opened a fresh Express boilerplate project in VS Code. The project already had the basic server setup, routes, and backend architecture. But something was still missing — the APIs were not fully structured yet.

Step 1 — Installing the Extension

The developer installed the Express API Actions Generator extension and started building the backend architecture step by step. 

Api/
 └── v1/

First came the Api folder, then the v1 version structure, followed by router configuration.

Step 2 — Creating the Journals Module

Inside the routes file, the developer clicked on AddTableName and entered: 

journals

Immediately, the extension generated the complete backend module automatically. 

Api/
 └── v1/
      └── journals/
           ├── controllers/
           ├── services/
           ├── routes/
           ├── end-points.js
           └── ...
Step 3 — The Need for ShowAll

Inside the data folder, many JSON files were stored. The developer now needed a functionality that could:

  • Fetch all records from JSON files
  • Return all journal entries
  • Send the response through an API endpoint

This functionality was called: ShowAll

Step 4 — Opening end-points.js

The developer opened: 

Api/v1/journals/end-points.js

At the top-right corner, a special menu appeared containing API commands. 

GET
POST
PUT
DELETE
SHOWALL

The developer clicked on: GET

Step 5 — Automatic Generation

Within seconds, the extension automatically generated:

  • Route registration
  • Controller logic
  • Service methods
  • JSON response handling
  • Storage retrieval flow

The generated endpoint now had the power to retrieve all records from the data folder automatically. 

GET /journals

Final Result

What once required manually writing routes, services, controllers, and response handlers... was now generated in just a few clicks using the Express API Actions Generator extension.
